# UX Audit — About

## Summary

The About section has a strong visual foundation (cyberpunk dark-mode aesthetic, fluid type sizing, responsive 2-column grid) but carries three issues that block production readiness: bio text contrast fails WCAG AA by a meaningful margin (~4.1:1 vs the 4.5:1 minimum), GSAP animations run unconditionally with no `prefers-reduced-motion` guard anywhere in the codebase, and the decorative section-number prefix is read aloud by screen readers as "zero one dot" with no suppression. Several lower-severity findings around semantic markup, redundant style declarations, and hardcoded color values are also documented below.

---

## Findings

| # | Severity | Dimension | Finding | File:Line | Suggested Fix |
|---|----------|-----------|---------|-----------|---------------|
| 1 | 🔴 Critical | Accessibility | `"01."` decorative prefix is inside a plain `<span>` with no `aria-hidden`. Screen readers announce it as "zero one dot About", which is confusing and noisy — the number carries no semantic meaning for AT users. | `About.jsx:39–44` | Add `aria-hidden="true"` to the `<span>` containing `"01."`. Pattern repeats in `Experience.jsx:131–135` and `Projects.jsx` — fix all three. |
| 2 | 🔴 Critical | Accessibility | Bio paragraph color `--color-text-muted` (`#64748b`) on section background `--color-bg-secondary` (`#0d0d14`) yields a contrast ratio of **≈ 4.1:1**, failing WCAG AA (requires 4.5:1) for normal-weight body text at 16px. | `About.jsx:65` / `index.css:9` | Lighten `--color-text-muted` to at minimum `#7b8fa8` (≈ 4.6:1 on `#0d0d14`). Verify with a tool such as WebAIM Contrast Checker before committing. |
| 3 | 🔴 Critical | Perceived Performance | Zero `prefers-reduced-motion` handling exists anywhere in the codebase — no CSS media query in `index.css` and no `window.matchMedia('(prefers-reduced-motion: reduce)')` check in any `useEffect`. The GSAP opacity + Y-translate stagger runs unconditionally, which can trigger vestibular disorders and violates WCAG 2.3.3 (AAA) and WCAG 2.1 guidance. | `About.jsx:12–28` / `index.css` (absent) | Inside the `useEffect`, gate the animation: `if (!window.matchMedia('(prefers-reduced-motion: reduce)').matches) { gsap.from(...) }`. Also add `@media (prefers-reduced-motion: reduce) { .about-animate { opacity: 1 !important; transform: none !important; } }` to `index.css` as a CSS-level fallback. |
| 4 | 🟡 Warning | Accessibility | `<section id="about">` has no `aria-labelledby` attribute. Without an accessible name, `<section>` is **not** exposed as a named `region` landmark in the accessibility tree — AT users navigating by landmarks will skip it entirely. | `About.jsx:31–34` | Add `aria-labelledby="about-heading"` to `<section>` and `id="about-heading"` to the `<h2>` at line 45. Apply same pattern to `Experience` and `Projects` sections. |
| 5 | 🟡 Warning | Accessibility | "Tech Stack" sub-label is a `<p>` element (not a heading), and the 8 technology tags are unsemantic `<span>` elements wrapped in a `<div>`. Screen readers read a flat stream of text with no list context, making it impossible to know how many items exist or where the list ends. | `About.jsx:72–93` | Promote the label to `<h3>` (or use `aria-label` on the list). Wrap tags in `<ul className="flex flex-wrap gap-3">` with `<li>` per tag. |
| 6 | 🟡 Warning | Responsiveness | The section declares `py-24 px-6` as Tailwind classes **and** `paddingTop: '6rem', paddingBottom: '6rem', paddingLeft: '1.5rem', paddingRight: '1.5rem'` as inline styles on the same element. Inline styles win in all browsers, making the utility classes dead code. On 375 px mobile, 96 px of vertical padding (top + bottom = 192 px!) creates excessive blank space relative to the section's 4-paragraph content. | `About.jsx:34–35` | Remove the inline padding props entirely and replace the Tailwind classes with `py-12 md:py-24 px-4 md:px-6` for a mobile-first responsive rhythm. |
| 7 | 🟡 Warning | Responsiveness | `min-h-screen` combined with `flex items-center` vertically centers the ~300 px content block within a 100 vh container on mobile. At 375 × 667 px this leaves ~180 px of empty space above and below, making the section feel unfinished and forcing more scrolling. | `About.jsx:34` | Replace `min-h-screen` with `md:min-h-screen` so it only applies on ≥ 768 px, or remove it entirely and rely on content height. |
| 8 | 🟡 Warning | Visual Consistency | The decorative divider line uses hardcoded `rgba(0,245,255,0.2)` instead of the design token `var(--color-accent-cyan)`. The same magic value appears in `Experience.jsx:144` and elsewhere. If the accent color is ever changed in `--color-accent-cyan`, these dividers won't update. | `About.jsx:52–53` | Replace `backgroundColor: 'rgba(0,245,255,0.2)'` with `backgroundColor: 'var(--color-accent-cyan)'` and add `opacity: 0.2` as a separate property, or use the Tailwind `opacity-20` utility. |
| 9 | 🟢 Suggestion | Visual Consistency | Tech stack tag colors are hardcoded hex strings in `profile.js` (e.g., `#61dafb`, `#8cc84b`). The component appends non-standard hex-suffix opacity values (`${color}10`, `${color}40`) for backgrounds and borders. This bypasses the design token system, produces inconsistent opacity rendering across browsers, and makes global theming impossible. | `profile.js:15–24` / `About.jsx:84–87` | Define technology brand colors as CSS custom properties in `index.css` (e.g., `--color-tech-react: #61dafb`) and reference via `var()`. Use `color-mix(in srgb, var(--color-tech-react) 10%, transparent)` for the background tint (or Tailwind opacity utilities). |
| 10 | 🟢 Suggestion | Perceived Performance | The 3 `.about-animate` elements animate with `stagger: 0.15s` and `duration: 0.8s`, so the last element finishes at **1.1 s** after the scroll trigger fires. On slow hardware or when users quickly scroll past, this can feel sluggish. The section header, bio column, and tech stack column each reveal independently, which may feel heavier than needed. | `About.jsx:16–17` | Reduce stagger to `0.08s` and duration to `0.6s`. Alternatively, group the header and body into a single tween and only stagger the tech tags individually for a snappier feel. |
| 11 | 🟢 Suggestion | Visual Consistency | The `max-w-5xl mx-auto w-full` container specifies both Tailwind utilities and redundant inline `style={{ maxWidth: '64rem', margin: '0 auto', width: '100%' }}`. This doubles maintenance surface. The same redundancy appears on the wrapper `<div>` in `Experience.jsx:128`. | `About.jsx:37` | Remove inline style props and rely solely on Tailwind classes, or vice versa. Adopt one consistent approach across all section wrappers. |
