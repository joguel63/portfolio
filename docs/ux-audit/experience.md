# UX Audit — Experience

## Summary
The Experience section has a solid semantic foundation (proper `<ul>/<li>` for achievements, CSS-var–driven colour tokens) but ships three gaps that need attention before launch: decorative timeline elements are not hidden from assistive technology, no `prefers-reduced-motion` guard exists on any GSAP animation, and the alternating left/right card animation direction becomes semantically meaningless on single-column mobile layouts. Visual consistency is largely good, though several inline style values duplicate design-token equivalents that already exist in `index.css`.

## Findings

| # | Severity | Dimension | Finding | File:Line | Suggested Fix |
|---|----------|-----------|---------|-----------|---------------|
| 1 | 🔴 Critical | Accessibility | Timeline center dot `<div>` is purely decorative but has no `aria-hidden="true"`; screen readers will announce the unlabelled element as it enters focus traversal | `Experience.jsx:17–26` | Add `aria-hidden="true"` to the dot `<div>` |
| 2 | 🔴 Critical | Accessibility | Timeline vertical line `<div>` is purely decorative but has no `aria-hidden="true"`; same screen-reader pollution risk as the dot | `Experience.jsx:149–152` | Add `aria-hidden="true"` to the line `<div>` |
| 3 | 🔴 Critical | Perceived Performance | GSAP `gsap.from('.experience-card', …)` runs unconditionally — no `prefers-reduced-motion` check; users who opt out of motion still get 0.8 s sliding/fading animation | `Experience.jsx:103–119` | Wrap with `gsap.matchMedia()`: run the full tween inside `"(prefers-reduced-motion: no-preference)"` and show cards immediately (opacity 1, x 0) inside the reduced-motion media query |
| 4 | 🟡 Warning | Accessibility | Period label is a plain `<span>` with no machine-readable time semantics; screen readers announce it as raw text without conveying that it is an employment date range | `Experience.jsx:51–56` | Replace `<span>` with `<time>` and add `aria-label="Employment period: …"`, e.g. `<time aria-label="Employment period: 2023 to present">` |
| 5 | 🟡 Warning | Accessibility | `<section id="experience">` has no `aria-labelledby`; landmark navigation in screen readers will announce it as an unlabelled region rather than "Experience region" | `Experience.jsx:122–126` | Add `aria-labelledby="experience-heading"` to `<section>` and `id="experience-heading"` to the `<h2>` |
| 6 | 🟡 Warning | Responsiveness | On mobile (<768 px) the layout collapses to a single column, but the GSAP `x` offset is still applied based on index parity (`i % 2 === 0 ? -60 : 60`); cards slide in from opposite horizontal directions even though they are stacked vertically, creating a jarring visual inconsistency | `Experience.jsx:106` | Use `gsap.matchMedia()` to apply `x: ±60` only at `md+` breakpoints; fall back to `y: 20` only on mobile |
| 7 | 🟡 Warning | Responsiveness | The card header row (`flex items-start justify-between`) places the `whitespace-nowrap` period label inline with the role title; on 375 px with a long role title the two elements compete for space, risk truncation or overflow | `Experience.jsx:36, 52–55` | Stack the period below the company name on small screens using `flex-col sm:flex-row` on the header row, removing `justify-between` at mobile breakpoint |
| 8 | 🟡 Warning | Visual Consistency | Card background `rgba(13,13,20,0.8)` and border `rgba(0,245,255,0.15)` are hardcoded magic values that duplicate existing tokens (`--color-bg-secondary: #0d0d14`, `--color-accent-cyan: #00f5ff`); token drift risk if palette changes | `Experience.jsx:31, 33` | Use `background-color: color-mix(in srgb, var(--color-bg-secondary) 80%, transparent)` and `border-color: color-mix(in srgb, var(--color-accent-cyan) 15%, transparent)` |
| 9 | 🟡 Warning | Responsiveness | Section padding is declared twice: once via Tailwind (`py-24 px-6`) and again via redundant inline `style={{paddingTop:'6rem', …}}`; inline styles win specificity, making Tailwind responsive overrides (`md:py-32`, etc.) silently ineffective | `Experience.jsx:125–127`, `13–14` | Remove the duplicate `style={{}}` padding/gap declarations; rely solely on Tailwind utilities |
| 10 | 🟢 Suggestion | Perceived Performance | A single `trigger: sectionRef.current` fires all card tweens the moment the section top hits 75 % of the viewport; if the section is tall (multiple entries), lower cards animate in before the user sees them — wasted motion and a long cumulative animation window | `Experience.jsx:110–114` | Replace with per-card `ScrollTrigger` using the card element itself as trigger (`trigger: card, start: 'top 85%'`) so each card reveals naturally as it scrolls into view |
| 11 | 🟢 Suggestion | Accessibility | `--color-text-muted: #64748b` on `--color-bg-primary: #050508` measures ≈ 4.67:1 — technically above 4.5:1 for body text, but the `text-xs` (12 px) period label needs the same 4.5:1 ratio with less visual tolerance; combined with the dark backdrop-filtered card, effective contrast may dip below threshold | `index.css:9`, `Experience.jsx:54` | Lighten muted text to `#8899b0` (≈ 6:1 on the dark background) or increase the period label to `text-sm` |
| 12 | 🟢 Suggestion | Visual Consistency | Achievement bullet `▸` (U+25B8) is a raw Unicode character, inconsistent with the design-system checklist directive "No emojis as icons — use SVG (Heroicons/Lucide)"; it also inherits muted colour from its parent `<li>` via an explicit override span, adding unnecessary markup | `Experience.jsx:73` | Replace with a 12 px Lucide `ChevronRight` icon (`<ChevronRight size={12} aria-hidden="true" />`) coloured with `var(--color-accent-cyan)` |
