# UX Audit — Contact

## Summary

The Contact section is visually coherent with the cyberpunk design system and correctly uses CSS custom properties for most colors, but it has three critical runtime blockers: all content is invisible when GSAP fails (opacity:0 with no fallback), a flex-direction inline style override forces 3 full-width buttons into a single row at 375 px causing horizontal overflow, and the email link never surfaces the actual address, locking out users without a mail client. Several accessibility gaps (missing focus-visible styles, unguarded aria on the section number, and a contrast ratio just below WCAG AA) lower the section's overall score.

## Findings

| # | Severity | Dimension | Finding | File:Line | Suggested Fix |
|---|----------|-----------|---------|-----------|---------------|
| 1 | 🔴 Critical | Accessibility | Email social link exposes only `mailto:` behind the label "Email" — users without a mail client have no way to see the address or copy it | `Contact.jsx:30` | Display the raw email address as visible text (e.g. `joguel63@email.com`) or add a "Copy" icon button next to the link |
| 2 | 🔴 Critical | Responsiveness | `style={{ flexDirection: 'row' }}` on the social-link wrapper always overrides the Tailwind `flex-col` class; at 375 px the three `w-full` buttons are forced into one row, causing horizontal overflow | `Contact.jsx:107` | Remove the inline `flexDirection` and let `flex flex-col sm:flex-row` handle breakpoints, or change the inline style to `flexDirection: window.innerWidth < 640 ? 'column' : 'row'` — preferably just delete the override |
| 3 | 🔴 Critical | Perceived Performance | Every `.contact-animate` element starts at `opacity: 0` via `gsap.from()`; if GSAP or ScrollTrigger fails to load the entire section stays invisible — title, description, and all social links are hidden | `Contact.jsx:38–49` | Use `gsap.fromTo()` with an explicit `to: { opacity: 1 }`, or set a CSS fallback `opacity: 1` on `.contact-animate` and rely on GSAP overriding it, so non-JS / GSAP-fail users still see the content |
| 4 | 🟡 Warning | Accessibility | `--color-text-muted` (`#64748b`) on `--color-bg-primary` (`#050508`) yields ≈ 4.39:1 contrast — just below the WCAG AA 4.5:1 minimum for 16 px normal body text used in the description paragraph | `Contact.jsx:101`, `index.css:6,9` | Lighten `--color-text-muted` to ≈ `#718096` (≈ 4.6:1) or increase the muted text size to 18 px+ (large text threshold is 3:1) |
| 5 | 🟡 Warning | Accessibility | Section number `04.` is rendered inside a `<span>` with no `aria-hidden="true"`; screen readers announce "zero four period Contact" — the decorative numeral pollutes the heading's accessible name | `Contact.jsx:83` | Add `aria-hidden="true"` to the `<span>` wrapping `04.`, matching the pattern recommended for sibling sections |
| 6 | 🟡 Warning | Accessibility | No `focus-visible` style is defined anywhere for `<a>` elements; Tailwind v4 preflight removes browser default outlines, leaving keyboard users with zero visible indicator when tabbing through social links | `index.css` (absent), `Contact.jsx:115` | Add a global rule: `a:focus-visible { outline: 2px solid var(--color-accent-cyan); outline-offset: 3px; }` in `index.css` |
| 7 | 🟡 Warning | Visual Consistency | Global `a:hover { text-decoration: underline }` in `index.css` fires on social link text during hover, drawing an underline that visually conflicts with the neon glow + border highlight effect already applied via `onMouseEnter` | `index.css:65–66` | Scope the underline rule away from button-style links: `.contact a:hover { text-decoration: none; }` or add `text-decoration: none` in the hover inline style handler |
| 8 | 🟡 Warning | Visual Consistency | Hardcoded `rgba(0,245,255, …)` and `rgba(139,92,246, …)` magic numbers are used in the radial gradient background and in hover border/shadow styles instead of `var(--color-accent-cyan)` / `var(--color-accent-purple)` — a future token change will not propagate here | `Contact.jsx:65`, `Contact.jsx:117–126` | Replace with CSS custom properties: `color-mix(in srgb, var(--color-accent-cyan) 4%, transparent)` for low-opacity fills, or extract opacity variants as new `@theme` tokens |
| 9 | 🟢 Suggestion | Perceived Performance | The background radial gradient `pulse` keyframe animation has no `@media (prefers-reduced-motion: reduce)` guard, violating WCAG 2.1 SC 2.3.3 and causing unwanted motion for vestibular-sensitive users | `Contact.jsx:64–75` | Wrap the `animation` inline style in a check: `animation: window.matchMedia('(prefers-reduced-motion: reduce)').matches ? 'none' : 'pulse 4s ease-in-out infinite'`, or move to CSS and wrap in `@media (prefers-reduced-motion: no-preference)` |
| 10 | 🟢 Suggestion | Perceived Performance | No back-to-top link or copyright year is present; as the last section of the page, Contact provides no navigation affordance back to the hero and gives no temporal indicator of freshness | `Contact.jsx:141–145` | Add a `<a href="#hero">↑ Back to top</a>` and a `© {new Date().getFullYear()} Miguel Muñoz` line below the tech attribution |
| 11 | 🟢 Suggestion | Accessibility | `aria-label={label}` duplicates the visible text content of each social link button — the label "GitHub" already serves as the accessible name via text content; the redundant `aria-label` is unnecessary though not harmful | `Contact.jsx:114` | Remove `aria-label={label}` from each `<a>`; the `aria-hidden="true"` SVG icon + visible text label is a correct accessible pattern without it |
| 12 | 🟢 Suggestion | Visual Consistency | `transition-all duration-300` is at the upper bound of the 150–300 ms range for micro-interactions; `transition-all` also animates layout-affecting properties unnecessarily | `Contact.jsx:115` | Replace with `transition-colors duration-200 ease-out` to target only color/border/shadow and keep interactions snappy |
